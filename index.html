Solution

the problem is that when adding video to a page, it looks really bad, and stying the iframe itself is hard in CSS, so were going to add the iframe in a containerand it's going to act like Inception the movie but in code. ;)

example 1, adding the video in an iframe and that, in it's own container in HTML ok?

<iframe width="560" height="315" src="https://www.youtube.com/embed/Hm3JodBR-vs" frameborder="0" allow="accelerometer; autoplay; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></iframe>

here is what the CSS portion of example 1 looks like. it's targeting the iframe or class of that container

.videoWrapper iframe { position: absolute; top: 0; left: 0; width: 100%; height: 100%; }

here is what the parent container looks like. notice that it's targeting the container as a whole, but not the class

.videoWrapper { position: relative; padding-bottom: 56.25%; /* 16:9 */ padding-top: 25px; height: 0; }
